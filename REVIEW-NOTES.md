# Review Notes

Optional background.

Key lessons from the live evolution of this setup:

- keep a separate source-of-truth layer instead of burying everything in heartbeat text
- use Gmail message search, not thread-only search
- check all relevant calendars before offering or booking time
- use helper scripts as the stable Google command surface instead of raw one-off CLI invocations
- keep one canonical live task system and let Todoist own it
- separate direct task CRUD from morning task prep
- treat the outreach tracker as a live CRM source of truth and update it in the same turn
- treat meeting notes as an operational signal source, not passive documents
- use a priority map to decide what matters
- use an auto-resolver policy to decide whether to act, draft, escalate, or ignore
- keep proactive update transport configurable instead of hardcoding one surface
- keep cron prompts thin and deterministic, and let skills hold the real procedure
- use a shared task-system contract so EA, BD, and task flows do not drift apart
- keep heartbeat thin and use it for opportunistic compiler/lint work, not as the only dependable trigger for recurring operations
